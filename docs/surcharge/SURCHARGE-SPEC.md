# 燃料サーチャージ 届出書・距離制 仕様メモ

本ディレクトリは、燃料サーチャージの**届出書**と**距離制（県庁間距離）**の元資料と派生データを
保管する。原本 xlsx は施主提供。

## ファイル

| ファイル | 内容 |
|---|---|
| `燃料サーチャージ届出書.xlsx` | 届出書の原本（`条件設定シート` + `燃料サーチャージ届出用紙`） |
| `県庁間距離.xlsx` | 県庁間距離マトリクス原本（47×47、`都道府県 県庁所在地` ラベル） |
| `kenchokan-distance.csv` | 上記から派生した 47×47 距離 CSV（UTF-8 BOM、`都道府県,県庁所在地,<47県>...`）。距離制計算 / CSV download・input の元データ |

## 届出書の前提条件（原本 `条件設定シート` / `燃料サーチャージ届出用紙` より）

- **基準価格**: 100 円/L（石油情報センターの週間統計データによる）
- **改定する刻み幅**: 5 円/L
- **改定条件**: 刻み幅 5.00 円/L の幅で軽油価格が変動した時点で、翌月から改定
- **廃止条件**: 軽油価格が 100.00 円/L を下回った時点で、翌月から廃止
- **計算式**:
  - 距離制運賃: `走行距離(km) ÷ 燃費(km/L) × 算出上の燃料価格上昇額(円/L)`
  - 時間制運賃: `平均走行距離(km) ÷ 燃費(km/L) × 算出上の燃料価格上昇額(円/L)`
- **端数処理**: 円単位に小数を**切り上げ**（ceil）

### 算出上の上昇額テーブル（段階方式）

代表価格 = 区間の刻み幅 **0.5 倍**点（区間下限 + 2.5）。上昇額 = 代表価格 − 基準価格。

| 適用時の軽油価格 (円/L) | 代表価格 | 上昇額 |
|---|---|---|
| ～ 100（基準価格以下） | 廃止 | － |
| 100 超 ～ 105 | 102.5 | 2.5 |
| 105 超 ～ 110 | 107.5 | 7.5 |
| 110 超 ～ 115 | 112.5 | 12.5 |
| 115 超 ～ 120 | 117.5 | 17.5 |
| 120 超 ～ 125 | 122.5 | 22.5 |
| 125 超 ～ 130 | 127.5 | 27.5 |
| 130 超 ～ 135 | 132.5 | 32.5 |
| …（5 円刻みで継続）… | | |
| 180 超 ～ 185 | 182.5 | 82.5 |

> 一般化: 区間 `(base + 5k, base + 5(k+1)]` の代表価格 = `base + 5k + 2.5`、上昇額 = `5k + 2.5`（k = 0,1,2,…）。

### 車種別燃費（km/L）— 有効期間つきマスタ

原本では `●●km/L`（**未確定**）。小型車 / 中型車 / 大型車 / トレーラー の 4 区分。
燃費は改定され得るため、単一値ではなく **「いつから・いつまで」（有効期間）つきのマスタ** として
持つ（#1）。実値は画面の「燃費マスタ」から CSV で入力する。

CSV 形式（`fuel-efficiency-sample.csv` 参照、UTF-8 BOM、1 行 1 レコード）:

```
車種C,車種名,燃費,有効開始,有効終了
04,大型車,3.5,2026-01-01,
16,トレーラー,3.0,2026-01-01,2026-03-31
```

- `車種C`: 運転日報明細の車種C（`MeisaiRow.sharuC`）と突合
- `燃費`: km/L（小数可、0 以下は不可）
- `有効開始`: `YYYY-MM-DD`（必須・含む）
- `有効終了`: `YYYY-MM-DD`（含む）。**空 = 無期限**。同一車種で期間が重なる場合は
  有効開始が新しい方を採用

計算時は売上年月日に応じて `resolveFuelEfficiency(entries, 車種C, 売上年月日)` で
その時点の燃費を引く（`src/fuel-efficiency.ts`）。`toEfficiencyLookup` で
`SurchargeMasters.fuelEfficiency`（関数版）にそのまま渡せる。D1 は `fuel_efficiency`
テーブル（`migrations/0002_fuel_efficiency.sql`）。

| 車種 | 8 時間制 | 4 時間制 |

## 計算エンジン (`src/surcharge.ts`) は届出書方式に統一済み（#11-C）

当初 `src/surcharge.ts`（Refs #4）は raw 価格差 + `round` だったが、**#11-C で届出書の正式方式
（段階テーブル + `ceil`）に統一**した:

```
上昇額         = surchargeIncrement(当月全国平均軽油, 基準価格, 刻み幅)
                 = 区間 (base+step·k, base+step·(k+1)] → step·k + step/2  (基準価格以下は 0)
行サーチャージ = ceil( 上昇額 × 距離km ÷ 燃費km/L )      (円単位切り上げ)
```

- `SurchargeMasters.priceStep` で刻み幅を注入（省略時 5 = 届出書既定）。
- `surchargeIncrement(monthly, base, step)` は export 済みの純粋関数。届出書用紙の段階テーブル
  生成（#11-B）からも同じ関数を使う。
- 旧 raw 差方式（#4）からの変更で、同一入力でも額が下がる（例: 軽油 130 円・226km・燃費 4.0 →
  旧 round 1695 → 新 ceil 1554）。これは段階テーブルが区間中点（130 → 区間 (125,130] の代表
  127.5、上昇額 27.5）を採るため。

## 県庁間距離の使い方

`src/surcharge.ts` の `SurchargeMasters.distanceKm`（キー `積地県\t卸地県`、対称）に
`kenchokan-distance.csv` を流し込む。CSV は download（マスタ確認用）/ input（編集・更新用）
の双方に使う。都道府県名（`北海道` 等）をキーにし、`地域ﾏｽﾀ.地域N` 正規化後の県名と突合する。
